1. Single page application - Bu web tetbiq ve ya web sehifedi hansiki bir html documentden ibaret olur butun web sehifelere uzluk kimi isledilen, ve js wasitesile sehifede deyiwiklik etmek olur. Bu web sehife ve ya web tetbiqdi hansiki bir sehife kimi yuklenir, ve istifadecinin isletmeyine gore dinamik olaraq mezmunu (kontenti) yenileyir. Her sehifeye kecid etdikde yuklenmek evezine SPA js vasitesile yegane sehifenin mezmununu (contentini) deyiwir, ve belelikle sehifeni daha hamar (плавный) ve suretli edir.
2. Virtual DOM - istifadəçi interfeysi (UI) yeniləmə prosesini asanlaşdırmaq üçün React kimi bəzi müasir kitabxanalarda istifadə olunan bir modeldir.
3.Real DOM - Real DOM brauzerin veb-səhifənin məzmununu görməsi və manipulyasiya etməsi üsuludur. Veb səhifəni açdığınız zaman brauzer bütün HTML kodunu ağaca bənzər struktura çevirir. Hər səhifə elementi (başlıqlar, düymələr, şəkillər və s.), hər bir mətn və atributlar (məsələn, rənglər, keçidlər) bu ağacın “budaqları” olur.
4. FrameWork - proqram təminatının işlənməsini sadələşdirmək üçün nəzərdə tutulmuş strukturlaşdırılmış platforma və ya alətlər və kitabxanalar dəstidir, o ümumi tapşırıqlar üçün əvvəlcədən qurulmuş həlləri özündə cəmləşdirir. Bir sozle developerin isini asanlawdirmaq ucun aletler toplusudur.
5. Reconciliation React - bu, virtual DOM-u yeniləmək və onu real DOM ilə sinxronlaşdırmaq prosesidir. Bu, React-in sürətli və səmərəli olmasını təmin edən əsas xüsusiyyətlərindən biridir.
6. Array.map - map() metodu massivin hər bir elementi üçün müəyyən edilmiş funksiyanın çağırılması nəticəsində yeni massiv yaradır.
7. Context api - bizim merkezde bir eded melumatimiz olur ve biz o melumati istediyim komponente oture bilirik. Təsəvvür edin ki, bir qrup komponentləriniz var və hər komponent eyni melumati bilməlidir Bu mesajı bir zəncir boyunca bir komponentden digərinə ötürmək əvəzinə Siz sadəcə olaraq hər kəsin görüb oxuması üçün işarəni (kontekst) divara asa bilərsiniz
8. UseEffect - Componentlerin 3 yawam dovru var: mount, update ve unmount. useEffect onlarin idare olunmasi ucun istifade olunur
9. Controlled input - tamamilə React tərəfindən idarə olunan bir input elementidir. Bu o deməkdir ki, input sahəsinin dəyəri komponentin state-de saxlanılır və input sahəsinə edilən hər hansı dəyişiklik bu vəziyyətin yenilənməsinə səbəb olur.
10. UnControlled input -React-də idarə olunmayan input elementidir. Bunun əvəzinə, input dəyəri birbaşa DOM-dan alınır, məsələn, ref.
11. React children - bu xususi bir propsdur hansiki daxili elementleri diger komponentlerin arasindan kecirmeye imkan verir.
12. Style.css & Style.module.css - Style.css de bir komponente style verdikde onun adinda olan butun komponentlerde deyiwiklik olur. Module.css de ise her bir komponente oz style-ni veririk
13.Package.json - bu bir falydir hansiki node.js projectlerinde istifade olunur ve projectin parametrlerini idare edir. Orda esasen project haqqinda melumatlar olur mes: projectin adi, versiyasi, muellifi. Datalar orda saxlanilir. Orda kitabxanalarda saxlanilir hansiki projectin islemesine komek olunur.
14. MUI - Material UI - Google-in dizayn prinsiplerine esaslanan React kitabxanasidir.
15. createTheme() - bu bir React ucun Material UI kitabxanasindan gelen funksiyadir hansiki onun komeyi ile tetbiqin temalarini deyiwdire bilersiz ve o deyiwikleri butun tetbiqde istifade etmek olacaq.
16. Stack - funksiya cagirilanda o stek yaddawda saxlanilir, funksiya iwini bitirende stackdan silinir. xususi bir componentdiki display flex i var
